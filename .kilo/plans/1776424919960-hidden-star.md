# Plan: Install Penpot MCP for Kilo CLI (Project-Level)

## Context

Gemini hallucinated `@penpot/mcp-server` — the actual package is **`@penpot/mcp`**. The Penpot MCP server runs as an **HTTP server** (not stdio), so Kilo must connect to it via `type: "remote"` pointing to `http://localhost:4401/mcp`.

Remote MCP isn't in production yet (planned ~v2.16), so we're using **Local MCP** mode only.

## Changes

### 1. Create project-level Kilo config: `.kilo/kilo.json`

Add an MCP server entry for Penpot. This keeps it scoped to this project only — no pollution of other projects' context.

```json
{
  "mcp": {
    "penpot": {
      "type": "remote",
      "url": "http://localhost:4401/mcp",
      "enabled": true,
      "timeout": 30000
    }
  }
}
```

**Why `type: "remote"`?** The Penpot MCP server exposes an HTTP endpoint (streamable HTTP at `/mcp`), not a stdio interface. Kilo's `type: "local"` spawns a stdio process, which doesn't match. `type: "remote"` connects to a URL, which is exactly what we need for `http://localhost:4401/mcp`.

**Why `timeout: 30000`?** Design file operations (export, high-level overview) can take longer than the default 10s.

### 2. No automatic server startup

The Penpot MCP server requires a **running Penpot browser session** to function (it proxies through your active browser tab). There's no way to auto-start it meaningfully — the user must:

1. Run `npx @penpot/mcp@stable` in a terminal (keep it running)
2. Open Penpot → load plugin from `http://localhost:4400/manifest.json`
3. Click "Connect to MCP server" in the plugin

If the server isn't running when Kilo starts, Kilo will log a connection failure but continue normally. The MCP tools simply won't be available until the server is up.

## User Instructions (post-install)

### Starting the MCP server

```bash
# In a separate terminal, start the Penpot MCP server:
npx @penpot/mcp@stable
```

### Connecting the Penpot plugin

1. Open https://design.penpot.app and open a design file
2. Go to **Plugins → Load from URL** and enter: `http://localhost:4400/manifest.json`
3. Run the plugin and click **Connect to MCP server**
4. Keep the plugin window open while working with AI agents

> **Browser note:** Some Chromium-based browsers block `https` → `http://localhost` connections. If the plugin can't connect, explicitly allow local network access or use Firefox.

### First prompts to try

- "List pages in this file."
- "Show all components on this page."
- "Analyze the structure of this design and summarize it."

### Stopping

- Press `Ctrl+C` in the terminal running the MCP server
- Disconnect the plugin in Penpot
