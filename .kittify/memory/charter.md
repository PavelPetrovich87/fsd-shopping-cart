## FSD Architecture Rules

### Layer Hierarchy (strict, never reverse)

```
app -> pages -> widgets -> features -> entities -> shared
```

### Import Rules

- Cross-slice imports use @/ absolute paths via index.ts public API
- No relative imports that escape the slice boundary
- All cross-slice imports documented in ARCHITECTURE.md Slice-Level Graph

### Story-First Convention

For shared/ui components:
1. Create ComponentName.stories.tsx FIRST
2. Write the component implementation to satisfy the stories
3. Stories are executable specifications

### Named Exports Only

- `import/no-default-export: 'error'` in ESLint
- Exceptions: config files (vite.config.ts, steiger.config.ts, eslint.config.js) and *.stories.tsx

### No className on Custom Components

- ESLint rule: `no-restricted-syntax` for JSXAttribute[name.name="className"]
- Does NOT apply to shadcn/ subdirectory
